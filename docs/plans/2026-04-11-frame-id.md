# Plan: Replace raw `int` frame indices with abstract `FrameId` type

## Context

`ManagedPointerSource.LocalVariable` and `.Argument` encode method frame identity as a raw `int` index into `ThreadState.MethodStates`. This makes the "frame index = array position" invariant implicit rather than type-enforced. A review comment asks us to make these abstract references resolved by the machine state, following the existing pattern of `ThreadId` and `ManagedHeapAddress`.

This is a mechanical, type-driven refactor. The F# compiler will catch every site that needs updating. No behavioral change.

## Int extraction pattern

At every site where we index into `MethodStates` with a `FrameId`, use inline destructuring:

```fsharp
let (FrameId idx) = frameId
state.MethodStates.[idx]
```

This follows the existing codebase style for `ThreadId`/`ManagedHeapAddress` (no `toIndex` helper module).

## What this does NOT do (deliberate scope limit)

- **No runtime dangling-frame detection.** Frames are never removed from `MethodStates` (only appended), so indices can't go stale in the current implementation. Detecting pointers that outlive their logical frame lifetime requires tracking "frame is alive" state — a separate, larger change. `FrameId` sets the foundation: once all resolution goes through typed handles, adding validation to the resolution point is straightforward.
- **No generation/version counter.** Same reasoning — the abstraction makes this a future possibility without further refactoring consumers.
